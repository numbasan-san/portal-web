generación de código intermedio y final

¿Qué es el código intermedio (IR)?

El código intermedio es una representación del programa situada entre el lenguaje fuente y el código máquina. Sus objetivos son:

Abstraer detalles del lenguaje y de la máquina para facilitar análisis y optimización.
Permitir portabilidad el front-end produce IR y el back-end solo necesita traducir IR a la ISA objetivo.
Proporcionar una estructura adecuada para aplicar transformaciones y optimizaciones.

Tipos comunes de IR

Árboles (AST extendidos)

Estructuras jerárquicas que representan expresiones y sentencias.

Código de tres direcciones

Secuencia lineal de instrucciones con temporales (ej. t1 = t2 + t3).

SSA (Static Single Assignment)

Forma en la que cada variable recibe asignación una única vez; facilita análisis de flujo de datos y optimizaciones.

Bytecode / VM IR

Representación para máquinas virtuales (por ejemplo, Java bytecode, CLR).

Generación del IR

Después del análisis sintáctico y semántico, el compilador traduce el AST a una o varias formas de IR. Este proceso puede incluir:

Normalización de expresiones
Transformación a formas canónicas
Introducción de temporales
Variables intermedias para cálculos
Descomposición de construcciones complejas
Simplificación de estructuras

Optimización sobre IR

Las optimizaciones se aplican mayoritariamente sobre IR porque:

  • Evitan repetir optimizaciones para cada arquitectura destino.
  • IR ofrece una abstracción conveniente para análisis (gráficas de flujo de control, def-use chains).

Ejemplos de optimizaciones:

Eliminación de código muerto
Propagación de constantes
Análisis de alcance de variables
Optimización de bucles (invariant code motion)
SSA-driven optimizations

Generación de código final (back-end)

La generación de código final consiste en transformar el IR optimizado en instrucciones concretas de la ISA objetivo. Tareas clave:

Tarea Descripción
Selección de instrucciones Mapear operaciones del IR a secuencias de instrucciones máquina.
Asignación de registros Decidir qué temporales viven en registros y cuáles en memoria; usar técnicas de spilling cuando hay falta de registros.
Organización de pila Generar prologs/epilogs de función, manejar parámetros y retornos según la convención de la plataforma.
Emisión de assembler/objeto Crear archivos ensamblador y luego invocar ensamblador/linker para producir ejecutable.
Nota: algunos compiladores generan directamente código máquina desde IR; otros generan ensamblador intermedio.
